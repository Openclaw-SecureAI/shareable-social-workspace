# Workflow

V2.3 workflow:
1. create a post package from a topic
2. generate the text and visual together
3. save the structured draft in `drafts/pending/`
4. review with the user
5. capture brand/visual feedback in the workspace brand notes
6. request revision and regenerate the package if needed
7. approve or reject with explicit semantic commands

Do not publish directly yet.
Publishing remains an adapter boundary for later phases.
